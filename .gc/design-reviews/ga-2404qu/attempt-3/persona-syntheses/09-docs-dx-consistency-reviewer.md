# Felix Moreau

**Persona verdict:** block

**Sources:** Claude, Codex, DeepSeek, Gemini

**Consensus findings:**
- [Blocker] Runtime state for retired Maintenance is still undecided, so docs, doctor output, and operator commands cannot be truthful yet. Reviewers agree `.gc/runtime/packs/maintenance` is not merely stale wording: `jsonl-export.sh`, `spawn-storm-detect.sh`, and `jsonl_archive_doctor_check.go` currently use it for JSONL archives, export state, push-failure counters, and storm ledgers. The design lists possible dispositions but does not choose read paths, write paths, doctor behavior, `gc doctor --fix` behavior, rollback behavior, or operator command examples.
- [Blocker] The canonical system-pack reference anchor is not concrete. The design names `docs/reference/system-packs.md`, but that file does not exist and is not registered in `docs/docs.json`. A shared Core/Gastown/retired-Maintenance vocabulary needs a real navigable page or an explicitly designated existing page.
- [Blocker] The docs inventory/lint gate is not authoritative enough. The hand-authored edit list contradicts live files, especially `docs/guides/shareable-packs.md`, and the proposed grep misses `.mdx`, generated schema/reference files, docs navigation, CLI reference, examples, scripts, and doctor/code strings. A short representative list cannot support repo-wide docs/DX consistency.
- [Blocker] The design's own canonical terminology is inconsistent. It alternates between "required host system pack" and "required system pack," while also proposing separate doctor, docs, tutorial, and pack-comment wording. This is exactly the drift the design is supposed to prevent.
- [Blocker] Public Gastown's `dog` patch and host-Core dependency semantics are unresolved but user-facing. Public Gastown currently patches `dog`; after migration `dog` comes from required Core, not an explicit import. The design must specify whether `[[patches.agent]] name = "dog"` resolves against required-but-not-imported system packs, whether Gastown defines its own dog worker, or whether Core import semantics change. Docs and tutorials cannot describe the new model until this is decided.
- [Major] Troubleshooting guidance has live Maintenance references beyond path strings. Reviewers call out "Maintenance skips GitHub gate checks," JSONL export prose, the `state_dir/packs/maintenance/jsonl-archive` command, and `GC_JSONL_MAX_PUSH_FAILURES` environment guidance. These need semantic rewrites tied to the runtime-state decision, not blind replacement.
- [Major] Operator-facing examples remain copy-paste traps. Reviews identify stale or soon-deleted paths in `shareable-packs.md`, `coming-from-gastown.md`, `migrating-to-pack-vnext.md`, `gc-start-walkthrough.mdx`, `docs/reference/cli.md`, `examples/swarm/city.toml`, `examples/dolt/pack.toml`, `examples/dolt/assets/scripts/port_resolve.sh`, and Gastown status-line scripts.
- [Major] First-run and tutorial acceptance criteria are too thin. The design mentions golden tests for first-run/tutorial text, but does not define the minimal-city, Gastown-template, and upgraded-city stories those tests assert. It also risks calling `dog` an agent when the UI presents it as a pool; tutorial wording should describe a Core-provided housekeeping worker pool if that is the exposed model.
- [Major] Order-name migration affects configuration and routing, not only wording. Maintenance-owned orders moving to Core may change qualifiers from `maintenance.*` to `core.*`, affecting `skip_orders`, `order_config`, event/operator output, and `orderroute.QualifyPool`. The docs need a chosen alias/rename/migration story backed by tests.
- [Major] The canonical wording matrix and allowlist rules are incomplete. The design correctly protects `[maintenance.dolt]`, but it must also preserve `gc.store.maintenance.*` event names and OpenAPI schema entries as Dolt/store-maintenance vocabulary. It also needs executable rules for generated docs, historical plans/specs/runbooks, examples, tests, pack comments, and legitimate role/configuration mentions.
- [Minor] Tutorial 07's built-in order list, generated CLI examples, and docs-reference examples need ownership-aware updates so minimal Core cities and Gastown-template cities do not appear to ship the same local packs or orders.

**Disagreements:**
- Claude gives `approve-with-risks`; Codex, DeepSeek, and Gemini block. Assessment: choose `block` because runtime-state paths, the missing reference anchor, inconsistent canonical wording, and dog/Core patch semantics must be resolved before consistent docs can be written.
- Claude emphasizes the improved governance shape: inventory, canonical wording, docs lint, and golden tests. Assessment: keep that shape, but make the inventory the enforceable source of truth and subordinate or delete the stale hand-picked edit list.
- Reviewers differ on whether JSONL path handling is a docs issue or a runtime migration issue. Assessment: it is both, but runtime disposition comes first; docs, doctor output, and troubleshooting commands should cite the chosen state table.
- The system-pack anchor could be a new page or an existing page. Assessment: either is acceptable, but the design must pick one and wire navigation, cross-links, and golden/docs-lint expectations accordingly.
- Some findings frame `dog` patch semantics as a Gastown boundary concern rather than docs/DX. Assessment: retain it here because unresolved config semantics directly determine tutorial wording, doctor diagnostics, and public-pack guidance.

**Missing evidence:**
- A runtime-state migration table for JSONL archive repos, export state, push-failure counters, spawn-storm ledgers, order-tracking state, `GC_PACK_STATE_DIR`, legacy read paths, current write paths, doctor behavior, fix behavior, commands, and rollback.
- A concrete canonical system-pack reference destination: new `docs/reference/system-packs.md` registered in `docs/docs.json`, or an existing registered page named as the owner.
- A single verbatim terminology matrix for required Core, provider system packs, public Gastown, retired Maintenance, stale generated directories, Core's housekeeping worker pool, store/Dolt maintenance, and preserved `gc.store.maintenance.*` event names.
- A generated inventory of all docs, examples, command help, schema docs, generated references, scripts, pack comments, doctor strings, code paths, `.mdx` files, and docs navigation containing retired Maintenance/Gastown local-pack terminology.
- A docs-lint allowlist classifying every retained hit as historical archive, migration fixture, legacy diagnostic test, generated output, still-valid store/Dolt maintenance terminology, or intentional configuration/role vocabulary.
- The target first-run story for minimal `gc init`, the Gastown wizard/template path, `gc status`, existing upgraded cities, stale generated directories, and Core's `dog` worker pool.
- The chosen `[[patches.agent]]` resolution behavior for public Gastown patching Core's `dog` without an explicit Core import.
- The order-qualified-name migration strategy for `maintenance.*` orders moved to Core: aliases, preserved qualifiers, config rewrites, or explicit manual migration.
- The scope of the Core role-name guard for Go source, pack asset filenames, prompts, formulas, skills, docs, examples, tests, and historical docs.
- A transition plan for `examples/gastown/` while public Gastown lands and in-tree pack sources are deleted.

**Required changes:**
- Add the runtime-state migration table and make troubleshooting, doctor messages, golden tests, and rollback docs cite it. Include dual-path or migration behavior for `jsonl_archive_doctor_check.go`, JSONL archive state, spawn-storm state, order-tracking state, `GC_PACK_STATE_DIR`, and legacy fallback paths.
- Make the canonical system-pack reference concrete: create `docs/reference/system-packs.md` and register it, or designate and update an existing registered page. It must define required Core, provider system packs, public Gastown, retired Maintenance, stale generated directories, Core's housekeeping worker pool, store/Dolt maintenance, and preserved `gc.store.maintenance.*` event names.
- Replace all non-canonical wording in the design itself before asking implementers to update docs. Pick one phrase such as "required host system pack" or "required system pack" and use it verbatim across design sections, doctor text, docs, tutorials, and pack comments.
- Promote the inventory command to the authoritative completion gate. Expand it to include `.mdx`, generated schema/reference JSON and TXT, docs navigation/indexes, CLI reference, examples, script comments, doctor/code strings, docs plans/runbooks, and generated OpenAPI/schema references; require every retained hit to be allowlisted by category.
- Refine the inventory search so it is useful in practice: anchor noisy terms such as `dog` and `Core` with word boundaries or split the inventory into targeted searches for imports, paths, doctor strings, generated references, examples, and store-maintenance terms.
- Replace stale examples and copy-paste traps, including `[imports.maintenance]` / `../maintenance` / `export = true` in `shareable-packs.md`, local `packs/gastown` includes, deleted in-tree Gastown links, `.gc/system/packs/gastown` walkthrough paths, `--include packs/gastown` CLI examples, and example comments/scripts in `examples/swarm` and `examples/dolt`.
- Resolve public Gastown's `dog` patch semantics and document the chosen Core dependency model. Add config-resolution tests showing whether required-but-not-imported Core agents can be patched, and update tutorial/operator wording to match.
- Define first-run/tutorial expectations for minimal, Gastown-template or wizard-created, and upgraded cities. Update Tutorial 01 to describe `dog` as a Core-provided housekeeping worker pool, and update Tutorial 07's order list by template/ownership.
- Decide order qualifier migration for moved Maintenance orders and add tests for skip lists, order config, event/operator output, and `orderroute.QualifyPool`; document alias or rewrite behavior in doctor/troubleshooting guidance.
- Bind doctor output, FixHint text, docs, tutorials, pack comments, generated CLI/config references, and preserved event terminology to one canonical wording matrix, with golden tests that assert shared terminology rather than only absence of old words.
- Add a transition plan for `examples/gastown/` and generated docs/schema regeneration so the repo does not temporarily point users or tests at deleted local pack sources.
