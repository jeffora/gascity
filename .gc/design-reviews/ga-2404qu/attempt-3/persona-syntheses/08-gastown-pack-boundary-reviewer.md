# Avery McAllister

**Persona verdict:** block

**Sources:** Claude, Codex, DeepSeek, Gemini

**Consensus findings:**
- [Blocker] The public Gastown-to-host-Core dependency contract is still implicit. Current Gastown imports `../maintenance`, while the design says public Gastown should not import Core because Core is a required host system pack. The design must choose and test the actual mechanism: explicit Core import, auto-included host Core that public Gastown may patch, or Gastown-owned replacement assets. This affects `dog`, maintenance formulas/orders, prompt fragments, `session_live`, Polecat parent formulas, named session behavior, and diagnostics when Core is absent.
- [Blocker] Maintenance retirement runtime behavior is under-specified. Reviewers identify concrete loader/materializer/cache paths that need migration decisions: `requiredBuiltinPackNames`, `builtinPackIncludes`, `publicSubpathForPack`, `syntheticPackLayouts`, synthetic cache validation, `MaterializeBuiltinPacks`, stale-pack pruning, config import resolution, and `legacyPublicPackForSource`. Without these decisions, existing cities can keep loading stale Maintenance, lose operator-edited files, or hit phantom public-pack aliases.
- [Blocker] Runtime state paths still point at Maintenance. Gemini flags `cmd/gc/jsonl_archive_doctor_check.go` as compiled Go that resolves jsonl-export state from `packs/maintenance`, and multiple scripts use `GC_PACK_STATE_DIR` fallbacks under `.gc/runtime/packs/maintenance`. The design must specify the new state paths, keep legacy fallback behavior for existing cities, and test both old and new schemes.
- [Blocker] The behavior-preservation inventory is not scan-derived and already misses live role-bearing assets. Claude found `spawn-storm-detect.sh` classified as plain Core even though it sends mail to `mayor/`; DeepSeek and Gemini flag TOML defaults, step descriptions, and existing Core assets with `mayor`, `deacon`, `witness`, and `polecat` references. The Core role scanner protects Core from pollution, but it does not prove Gastown-side behavior is preserved when a missed branch is stripped.
- [Blocker] Test migration coverage is too narrow for deleting in-tree Gastown and Maintenance sources. The matrix covers `gastown_test.go` and `maintenance_scripts_test.go`, but reviewers identify sibling behavior tests and public-pack surfaces that need replacement coverage: commands/status, doctor checks, provider overlays, prompt fragments, tmux theme/keybindings, cycle behavior, operational awareness, PreCompact overlays, branch pruning, shutdown dance, Polecat compose, detector/requester paths, and review checks.
- [Blocker] The Core embed and registry transition must be atomic or explicitly compatible between steps. Moving Core out of `internal/bootstrap/packs/core`, retiring Maintenance from the registry/required-pack set, updating doctor/import handling, and deleting in-tree sources can leave duplicate Core embeds, stale Maintenance aliases, or moved-asset references if they land separately without compatibility gates.
- [Major] The Core maintenance-agent surface is not specified enough to be a public contract. The design must decide which `dog` agent fields are Core defaults, whether `fallback = true` changes all Core-only cities, how Gastown patches dog theming/work directories, and how renamed maintenance-agent cases keep warrants and shutdown routing coherent.
- [Major] Prompt, template-fragment, and `session_live` ownership remain unresolved. The monolithic `propulsion.template.md` contains definitions for multiple Gastown roles, while the dog prompt references `propulsion-dog`; `architecture`, `following-mol`, tmux theme/keybinding hooks, and named session declarations need explicit Core/Gastown/split dispositions and template-resolution tests.
- [Major] Several boundary assets still need final ownership decisions: the Codex overlay, `mol-review-quorum`, Core formulas that mention Gastown roles, Polecat base/work formula split, Dolt dependencies on maintenance formulas/scripts, and role-themed tmux Go APIs.
- [Minor] Legacy directory, stale import, and synthetic cache behavior need operator-facing diagnostics. Existing `.gc/system/packs/gastown` and `.gc/system/packs/maintenance` directories, public source aliases, local `examples/gastown/packs/*` imports, and retired state directories need explicit report/fix/ignore semantics.

**Disagreements:**
- Claude gives `approve-with-risks`; Codex, DeepSeek, and Gemini block. Assessment: choose `block` because the unresolved host-Core contract and runtime retirement paths directly affect `gc start`, `gc init`, config loading, doctor behavior, and existing-city upgrades.
- Claude treats the asset inventory as strong except for `spawn-storm-detect`; DeepSeek and Gemini argue the missed role sites show a structural flaw. Assessment: the inventory can remain the organizing artifact, but its completeness must be proven by scanner output rather than prior classification labels.
- Codex allows a stable Core `dog` name if it is named and tested as a contract; DeepSeek and Gemini question dog prompt fragments, TOML defaults, patch semantics, state paths, and `fallback = true`. Assessment: Core may own a configurable maintenance agent named `dog` only after the design specifies role-neutral prompt/templates, field defaults, state paths, and renamed-agent behavior.
- Reviewers differ on whether public Gastown should import Core explicitly or rely on auto-included host Core. Assessment: either can work, but the design must pick one, prove `[[patches.agent]]` semantics for the chosen path, and add failing diagnostics/tests for hosts that do not provide the required Core surfaces.
- The design says stale Maintenance/Gastown directories can be ignored, while reviewers show config resolution, materialization, pruning, script state, doctor checks, and source aliasing may still touch them. Assessment: treat "ignored" as unproven until real loader/materializer/cache/state tests cover stale directories and imports.

**Missing evidence:**
- Exact public `gascity-packs` prerequisite: repo/PR or branch, immutable commit, inventory path, CI command, required artifacts, and the `PublicGastownPackVersion` value.
- A declared public Gastown contract naming required Core surfaces: `dog`, maintenance formulas/orders, prompt fragments, scripts, Polecat parents, warrant/requester routes, provider hooks, doctor checks, commands, and review-flow surfaces.
- A scan output for every `gc mail send <role>`, `gc session nudge <role>`, TOML role-name default, role-bearing step description, prompt fragment reference, role-themed Go API, and runtime state path across Core/Core-renamed/Gastown-bound assets.
- A disposition for `cmd/gc/jsonl_archive_doctor_check.go`, `GC_PACK_STATE_DIR`, jsonl-export state, spawn-storm state, and stale `.gc/runtime/packs/maintenance` state after Maintenance retirement.
- A complete test-by-test migration table covering all current `examples/gastown/*.go` behavior tests, public-pack command and doctor tests, provider overlay tests, packlint/parser coverage, and current tests that assert Maintenance auto-inclusion.
- Integration proof that host Core plus pinned public Gastown, with no Maintenance pack, loads and exercises named behavior rather than only import wiring.
- Explicit handling for `publicSubpathForPack("maintenance")`, `requiredBuiltinPackNames`, `builtinPackIncludes`, `MaterializeBuiltinPacks`, stale-pack pruning, config import resolution, synthetic cache validation, and `legacyPublicPackForSource`.
- A decision for `internal/bootstrap/bootstrap.go`'s `//go:embed packs/**` and the future of `internal/bootstrap/packs/core` test fixtures or compatibility shims.
- Disposition of `dog` prompt templates, `propulsion-dog`, `architecture`, `following-mol`, Gastown `session_live` hooks, named session entries, and `dog` agent fields including fallback and pool limits.
- A release matrix case for new binary plus stale local Gastown/Maintenance imports and stale `.gc/system/packs/*` or `.gc/runtime/packs/maintenance` directories.

**Required changes:**
- Choose the public Gastown dependency mechanism for Core, document it as a contract, and add config/doctor diagnostics plus packcompat tests for host Core present, no Maintenance present, and host Core absent.
- Specify and test Maintenance retirement in runtime code: required-pack list, built-in includes, public-source aliases, synthetic cache validation, stale directory handling, stale import resolution, script state paths, doctor fix/report behavior, and operator-edit preservation.
- Replace classification-driven behavior preservation with scan-derived inventory. Fix `spawn-storm-detect` classification, include TOML defaults and step descriptions in the role scanner, enumerate existing Core role references, and require every role notification/nudge/default to become configurable in Core or preserved by a public Gastown inventory row.
- Expand the test migration matrix to all current Gastown behavior tests and public-pack surfaces, including commands/status, doctor/check-scripts, tmux theme/keybindings, cycle behavior, operational-awareness prompts, PreCompact overlays, branch pruning, shutdown dance, Polecat compose, and review checks.
- Specify dog prompt/template/session-live/named-session/agent-field ownership and add tests for default public Gastown load, renamed maintenance-agent behavior if supported, dog theming, warrants, shutdown routing, and cross-pack `mol-polecat-work` parent resolution.
- Resolve `review`/`split` ownership items before registry/source deletion, including Codex overlay, `mol-review-quorum`, Polecat base/work split, Dolt maintenance dependencies, monolithic `propulsion.template.md`, and role-themed tmux Go APIs.
- Resolve the bootstrap embed migration and require atomic landing of Core move, Maintenance registry removal, doctor/import changes, and in-tree source deletion, or define explicit intermediate compatibility tests for each step.
- Add lint/golden gates rejecting `[imports.maintenance]`, `../maintenance`, stale Maintenance-layer comments, retired local-pack guidance, and unintended stale Maintenance state references outside narrow migration-history allowlists.
