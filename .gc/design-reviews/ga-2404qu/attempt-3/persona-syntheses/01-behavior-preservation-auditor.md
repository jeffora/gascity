# Nadia Volkov

**Persona verdict:** block

**Sources:** Claude, Codex, DeepSeek V4 Flash (available as `01-behavior-preservation-auditor_gemini.md`)

**Consensus findings:**
- [Blocker] The behavior-preservation gate is still not complete or bidirectional. Reviewers agree that the design checks selected inventory rows or selected old tests, but it does not derive the inventory from every behavior-bearing asset and test. This leaves moved, split, generalized, helper-dependent, or currently untested behavior able to disappear without a required row or approved intentional-removal record.
- [Blocker] The gate does not block weakening-in-place. A row can record a `semantic delta`, and old assertions can be edited rather than removed, without a machine-enforced approval rule for trigger, requester, detector, route metadata, notification target, transport, script branch, state path, identity, or prompt-fragment changes.
- [Blocker] In-script Gastown escalations and identities have no concrete preservation path. `reaper.sh` and `jsonl-export.sh` currently hardcode mayor mail, deacon nudges, and the `Gas Town Daemon <daemon@gastown.local>` commit identity. The design sends the script/formula/order set toward Core without naming Core variables, Gastown rebinding assets, or behavior tests proving the original escalation and identity semantics still hold.
- [Blocker] Cross-pack dependencies are under-specified. The public Gastown pack depends on `imports.maintenance`, the `[[patches.agent]] name = "dog"` patch, helper script paths such as `status-line.sh -> _bd_trace.sh`, session-live hook scripts, duplicated template fragments, and named sessions. The design does not enumerate old owner, new owner, replacement mechanism, compatibility state, and test for these dependencies.
- [Major] Existing and proposed tests are existence-oriented where trigger-equivalence is required. Tests that prove a formula, detector, requester, notification, route, order, or helper "exists" do not prove the same trigger produces the same observable effect through the public Gastown pack with host Core present and no Maintenance pack.
- [Major] The high-risk inventory is narrower than the requirements map and source tree. Reviewers called out omitted or under-specified assets and behaviors including `reaper.sh`, `jsonl-export.sh`, `mol-do-work`, `mol-shutdown-dance`, `dolt-target.sh`, `_bd_trace.sh`, `status-line.sh`, tmux helper scripts, `[[named_session]]` entries, the `dog` agent configuration, acceptance tests, and `publicSubpathForPack`.
- [Major] The rollout and pinning plan can give false confidence. If the packcompat gate is introduced before `PublicGastownPackVersion` points at the immutable public Gastown commit, it validates the old pack during intermediate slices; the old-binary/new-pack state where Maintenance is still embedded but the public Gastown pack has dropped `imports.maintenance` is also not specified.

**Disagreements:**
- There is no verdict disagreement: Claude, Codex, and DeepSeek V4 Flash all block.
- The reviewers emphasize different layers of the same failure. Claude focuses on concrete escalation loss in scripts and on semantic-delta enforcement. Codex focuses on the inventory schema, pinned-pack CI gate, and behavior-specific tests. DeepSeek V4 Flash focuses on cross-pack dependencies, intermediate-state compatibility, helper paths, duplicated fragments, registry aliases, acceptance tests, and named-session behavior.
- Claude treats the reaper/jsonl issue mainly as missing Gastown re-hosting for Core-generalized scripts, while DeepSeek V4 Flash treats adjacent issues as missing dependency and state-path inventory. The synthesis assessment is that both are required: Core needs role-neutral variables and correct runtime paths, and Gastown needs explicit rebinding plus tests for the preserved Gastown behavior.

**Missing evidence:**
- A source-derived behavior inventory that scans pack files and tests for `gc mail send`, `gc session nudge`, `PACK_DIR`, `PACK_STATE_DIR`, `GC_PACK_STATE_DIR`, `GC_SYSTEM_PACKS_DIR`, role literals, daemon identity, `session_live` hooks, `imports.*`, `[[patches.agent]]`, `[[named_session]]`, route metadata, detector/requester entries, and behavior-bearing prompt fragments.
- A precise definition of a passing inventory row, including required old witness, new public-pack witness, immutable landing commit, consuming Gas City pin, and approved-loss marker for every semantic delta.
- A policy naming who approves intentional semantic deltas, where that approval is recorded, and which CI check fails if approval is missing.
- Executable tests for reaper/jsonl anomaly, spike, push-failure, mayor mail, deacon nudge, daemon commit identity, shutdown-dance requester/detector behavior, branch pruning, Polecat flows, `gc-dispatch` examples, and notification replacement semantics.
- A Gastown cross-pack dependency inventory for `imports.maintenance`, `[[patches.agent]] name = "dog"`, script-to-pack helper paths, `session_live` hook scripts, duplicated template fragments, named sessions, and the full `dog` agent configuration.
- A compatibility matrix entry for the intermediate state where the old Gas City binary still requires Maintenance while the public Gastown pack has already dropped or replaced the Maintenance import.
- A migration entry and tests for `publicSubpathForPack`, acceptance tests that assert Maintenance paths, `dolt-target.sh` path resolution, and the `GC_SYSTEM_PACKS_DIR` contract.
- The exact pin used by the packcompat gate at the time the gate is introduced.

**Required changes:**
- Replace the two-file deletion gate with a source-discovery-based old-behavior manifest. Every moved, split, generalized, retired, or helper-dependent behavior-bearing asset must produce an inventory row or an approved intentional-removal row, regardless of whether an existing test already covers it.
- Make inventory-row validation enforce semantic equivalence. Fail on unapproved deltas to trigger, requester, detector, route metadata, notification target, transport, script branch, prompt fragment, state path, identity, exit behavior, or once-only/throttle behavior; do not only fail on removed test files.
- Specify the reaper/jsonl preservation mechanism. Name the Core variables for escalation recipient, deacon/session nudge target, and commit identity; name the Gastown asset that rebinds them; and add public-pack tests proving the same anomaly/spike/push-failure behavior and daemon identity semantics. If any behavior is dropped, require explicit intentional-removal rows and operator release notes.
- Upgrade Gastown tests from existence checks to trigger-equivalence checks. For high-risk flows, tests must exercise the original trigger through the public Gastown pack with host Core and assert the observable effects, including recipient, transport, route metadata, requester/detector resolution, state writes, and prompt fragments where applicable.
- Add a required cross-pack dependency inventory. Cover `imports.maintenance`, the `dog` patch against a Core-provided base agent, script helper path changes such as `status-line.sh -> _bd_trace.sh`, `dolt-target.sh`, session-live hook dependencies, duplicated template fragments, `[[named_session]]` entries, and full `dog` lifecycle configuration, each with old behavior, new behavior, slice, and test.
- Align rollout and pinning. The packcompat gate must validate the immutable slice-1 public Gastown commit as soon as the gate lands, either by bumping `PublicGastownPackVersion` then or by using an explicit test pin. Add the missing intermediate-state compatibility row and state whether it is deployable or prevented.
- Add explicit migration tasks for `publicSubpathForPack`, acceptance tests that currently assert `packs/maintenance` paths, and any registry/cache alias behavior that still treats Maintenance as a public pack.
